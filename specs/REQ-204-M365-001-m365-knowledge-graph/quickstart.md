# Quickstart: Enterprise Knowledge Graph from Microsoft 365

**Phase**: 1 | **Audience**: frontend integrators, anyone standing up the backend locally for the first time

## 1. Prerequisites

- Go 1.22+
- Rust 1.75+ (for `llm-svc` ‚Äî see spec.md ¬ß3.5)
- PostgreSQL 14+ (locked decision ‚Äî not SQLite; see memory `req204-mergeassistant-critical-constraints` and spec.md provenance note)
- Neo4j 5.x
- `llm-svc` running locally (or reachable over gRPC) ‚Äî the Go backend no longer talks to any LLM provider directly (spec.md ¬ß3.3/¬ß3.5, amendment 2026-07-11)
- (Optional for local dev) A Microsoft Entra ID app registration ‚Äî otherwise use `M365_AUTH_MODE=jwt_demo`

## 2. Environment setup

**Go backend** (`backend/.env`) ‚Äî copy from plan.md / spec.md ¬ß12. Note: as of the gRPC amendment, Go holds **no** LLM provider credentials ‚Äî those moved to `llm-svc` (¬ß12.1):


**`llm-svc`** (`llm-svc/.env`) ‚Äî copy from spec.md ¬ß12.1:

```bash

LLM_API_KEY=<api-key>
LLM_MODEL=gpt-4o-mini
LLM_EMBED_MODEL=text-embedding-3-small
BRAIN_LOCAL_PROVIDER=qwen3-8b-q4
BRAIN_FALLBACK_TO_CLOUD=true
```

**Never hard-code these** ‚Äî CLAUDE.md ¬ß8 flags hard-coded hosts/ports as a common mistake to avoid.

## 3. First run (backend)

```bash
cd backend
go mod tidy
go run ./cmd/server            # runs migrations from internal/metadata/schema.go on startup
```

Verify: `curl http://localhost:8080/api/stats/overview` should return zeroed counts (no data ingested yet).

## 4. Connect an M365 source

```bash
curl -X POST http://localhost:8080/api/m365/connect \
  -H "Authorization: Bearer <token>" \
  -d '{"name":"Engineering OneDrive","type":"onedrive","tenant_id":"<tenant>","config":{"site_id":"<site>"}}'
```

Then trigger a sync:

```bash
curl -X POST http://localhost:8080/api/m365/sync -H "Authorization: Bearer <token>"
```

Watch progress over WebSocket (`ws://localhost:8080/ws?token=<token>`, event `sync_progress`) ‚Äî see contracts/api.md.

## 5. Ask a question (once ingestion + extraction has run at least once)

```bash
curl -X POST http://localhost:8080/api/knowledge/query \
  -H "Authorization: Bearer <token>" \
  -d '{"query":"Who is working on the Phoenix project?"}'
```

Expected response shape per contracts/api.md `POST /api/knowledge/query`: `answer`, `sources[]`, `entities[]`, `intent`, `latency_ms`. p95 latency target is ‚â§30s (plan.md Performance Goals).

## 6. Frontend integration

The frontend (`Frontend/` or a separate repo per plan.md) talks to the backend via:
- `VITE_API_BASE_URL` / `VITE_WS_URL` env vars (CLAUDE.md ¬ß7 ‚Äî never hard-code `localhost:8080`)
- TanStack Query hooks for all server state (`useKnowledgeQuery`, `useEntities`, `useFeedback`) ‚Äî never Zustand for server data (CLAUDE.md ¬ß6)
- The shared `useWebSocket` hook for real-time sync/query progress ‚Äî never instantiate `new WebSocket(url)` directly

Minimal integration test scenario (maps to spec.md ¬ß16 E2E acceptance flow, steps 1-7):
1. Log in (`/api/auth/login`) ‚Üí store JWT.
2. Configure a connection (`/api/m365/connect`).
3. Trigger sync (`/api/m365/sync`), observe `sync_progress` WebSocket events.
4. Poll `/api/entities?type=document` until non-empty.
5. Poll `/api/entities?type=person` / `?type=project` until non-empty (confirms NLP extraction ran).
6. Call `/api/graph/nodes` and `/api/graph/edges` ‚Äî confirm graph populated.
7. Call `/api/knowledge/query` with a question about ingested content ‚Äî confirm a cited answer comes back.

## 7. Independence check (MergeAssistant)

This repo is a standalone extraction (2026-07-11) from the parent MiniRag monorepo and does not itself contain `src/MergeAssistant/`, `src/Backend/`, or `src/Frontend/` ‚Äî those commands only apply inside the parent monorepo, where this feature originated as `src/m365-knowledge-graph/`. Before committing any change under `backend/` in *this* repo, confirm no accidental coupling was introduced:

```bash
grep -r "MergeAssistant" backend/ || echo "clean"
```

Must print `clean` ‚Äî see memory `req204-mergeassistant-critical-constraints`. (The reverse check, confirming the parent repo's `src/MergeAssistant/` has no coupling back into `src/Backend`/`src/Frontend`, is only relevant in the parent monorepo, not here.)
