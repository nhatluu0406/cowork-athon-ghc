File Work Review Deep Dive

File Work Review and Before/After Diff is currently PARTIAL PASS. The service and UI work exists, but the live packaged A-L acceptance run is not complete.

1. Permission pendingBefore snapshot
UI requests a bounded service-owned snapshot before an approved mutation lands. Snapshot includes SHA-256 hash, size, mtime, truncation state, and redaction decision.
2. MutationRuntime writes
File creation, modification, or deletion proceeds only through the existing permission boundary. Attachment context is separate from runtime file reads.
3. File eventAfter snapshot
On file_mutation, the UI captures after state with a short retry if disk flush timing needs it.
4. ArtifactBuild review
The service builds a persistent artifact with before, after, unified diff, truncation metadata, and secret redaction behavior.
5. PersistConversation activity
Review artifacts persist with conversation activity and remain visible after relaunch without re-reading current disk content to recreate history.
