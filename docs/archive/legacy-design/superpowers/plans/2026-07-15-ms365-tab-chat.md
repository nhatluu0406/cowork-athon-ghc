# MS365 Tab Chat (P5.6) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Chat thật trong tab Microsoft 365: session single-turn + streaming + transcript ephemeral, tự đăng ký/thu hồi `session-scope`, pill write-mode chuyển về composer tab.

**Architecture:** Theo spec `docs/superpowers/specs/2026-07-15-ms365-tab-chat-design.md` (căn cứ khảo sát có file:line — đọc spec trước). Thuần UI: controller tách khỏi DOM (`ms-chat-controller.ts`) + mở rộng `ms-assistant-view.ts` + 1 method client mới. KHÔNG sửa service.

**Tech Stack:** TypeScript strict, happy-dom cho view test, `node --import tsx --test`.

## Global Constraints

- **Commit only, KHÔNG push.** Không sửa `service/src/**` (route P5.5 đã đủ).
- **Thứ tự send bất biến**: `createSession` → `setMs365SessionScope(id, true)` → `startEvStream` → `sendSessionMessage`. Scope đăng ký TRƯỚC send — unit test assert thứ tự.
- **Revoke scope** ở: terminal, cancel, MS365 disconnect, reset tab. Revoke lỗi mạng → nuốt có chủ đích (scope in-memory service-side, app restart tự sạch) nhưng vẫn stop stream.
- Session single-turn: MỖI lượt gửi = session mới (409 `session_completed` nếu tái dùng — không tái dùng).
- Dispatch: `planDispatchPrompt(messages, [], prompt, undefined, [], true)` — tab luôn `ms365Connected=true`; budget fail → hiện lỗi, KHÔNG tạo session.
- Transcript ephemeral in-memory; không đụng conversation store.
- UI không gọi filesystem/credential; không secret trong state/DOM/log; copy user-facing tiếng Việt.
- Pill: sau slice này cowork composer KHÔNG còn pill (gỡ instance + `refreshMs365WriteModePill` + wiring cũ); pill tab chỉ hiện khi `connectionState === "connected"`.
- Test UI: `cd app/ui && node --import tsx --test "tests/<file>.test.ts"`. Typecheck: `npm run typecheck` (repo root).

**Interfaces (Produces):**

```ts
export interface MsChatMessage { role: "user" | "assistant"; content: string; pending?: boolean; error?: string }
export interface MsChatState {
  readonly messages: readonly MsChatMessage[];
  readonly phase: "idle" | "running" | "error";
  readonly sessionId: string | null;
  readonly errorMessage: string | null;
}
export interface MsChatDeps {
  preflight(): { canSend: boolean; message: string };          // bọc assessSendPreflight từ app-shell
  workspaceId(): string | null;
  createSession(input: { workspaceId: string; title: string }): Promise<{ id: string }>;
  setSessionScope(sessionId: string, enabled: boolean): Promise<void>;
  sendMessage(sessionId: string, text: string): Promise<{ accepted: boolean; reason?: string }>;
  cancelSession(sessionId: string): Promise<void>;
  startStream(sessionId: string, onView: (view: { text: string; terminal: string | null }) => void): { stop(): void };
  buildDispatch(prior: readonly MsChatMessage[], prompt: string): { ok: true; text: string } | { ok: false; message: string };
  onStateChange(state: MsChatState): void;                      // view re-render hook
}
export interface MsChatController {
  state(): MsChatState;
  send(prompt: string): Promise<void>;
  cancel(): Promise<void>;
  reset(): Promise<void>;          // revoke + clear transcript
  onDisconnected(): Promise<void>; // revoke + stop stream, giữ transcript, phase idle
}
export function createMsChatController(deps: MsChatDeps): MsChatController;
```

Ghi chú thiết kế: `buildDispatch` là seam bọc `planDispatchPrompt(prior→ConversationMessage[], [], prompt, undefined, [], true)` — controller không import dispatch-plan trực tiếp (test thuần logic); adapter thật viết ở Task 3 trong app-shell. `startStream` seam bọc `startEvStream` — map `view` thật (text qua `sanitizeAssistantForDisplay`/finalize helper, terminal) ở Task 3.

- [ ] **Step 1 (RED):** Viết `app/ui/tests/ms-chat-controller.test.ts` — fake deps ghi `calls: string[]` (nhãn `create`/`scope:true:<id>`/`stream:<id>`/`send:<id>`/`scope:false:<id>`/`cancel:<id>`):
  - send hạnh phúc: thứ tự calls đúng bất biến; user message append trước, assistant pending bubble xuất hiện; phase running.
  - onView cập nhật assistant content (replace); terminal → `scope:false` + stream stop + phase idle + pending clear.
  - preflight fail → 0 call, errorMessage set.
  - buildDispatch fail (budget) → 0 session, lỗi hiển thị.
  - `sendMessage` không accepted → revoke + lỗi trung thực (không treo running).
  - cancel giữa chừng → `cancel:<id>` + `scope:false` + stream stop, assistant bubble đánh dấu hủy.
  - reset → revoke (nếu đang có session) + messages rỗng.
  - onDisconnected → revoke + stop, transcript GIỮ NGUYÊN.
  - send khi phase running → từ chối (no-op có thông báo) — không session chồng.
  - revoke ném lỗi mạng → không throw ra ngoài, stream vẫn stop.
