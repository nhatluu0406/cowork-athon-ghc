# MS365 P2-A History Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist MS365 chat messages into their conversation record and add a sidebar in the "Trợ lý AI" tab to list, open (view + continue), and start MS365 conversations.

**Architecture:** Pure UI work reusing existing backend. `Ms365ChatController` gains a `conversationId` getter plus `resetConversation()`/`adoptConversation(id)` so app-shell can start a new conversation or open an old one. The MS365 send/stream path calls the existing `appendConversationMessage` (A0 — the prerequisite that makes history non-empty). A sidebar in the assistant tab lists `surface=ms365` conversations via the existing `listConversations(query, "ms365")`, and selecting one loads its transcript via `getConversation` into the existing `ms365Messages` array.

**Tech Stack:** TypeScript (NodeNext ESM, `.js` import specifiers), vanilla-TS renderer, `node:test` via tsx.

## Global Constraints

- MS365 messages MUST be persisted via `appendConversationMessage(conversationId, role, text)` — user after `send()` (once `conversationId` exists), assistant when the stream terminal is `"completed"` (only when `conversationId != null` and text is non-empty). Persist is best-effort: a persist failure must NOT break the chat turn or the in-memory transcript. (spec §4 A0, §5)
- The controller keeps ONE live session per open conversation. `resetConversation()` → `conversationId=null` + reset session/stream (new conversation on next send). `adoptConversation(id)` → `conversationId=id` + reset session/stream (old conversation; next send creates a NEW runtime session + scope-allow and appends to the old id — continuation). Neither revokes scope. (spec §2, §4 A1)
- Opening an old conversation loads its transcript from `getConversation(id)` into `state.ms365Messages`; the controller does NOT re-create the conversation on the next send (guard: `conversationId != null`). (spec §4 A1)
- Sidebar + chat only render when `connectionState === "connected"`; unconnected shows the existing "Chưa kết nối" card. (spec §2)
- MS365 conversations (`surface=ms365`) never appear in the Cowork sidebar (already true from P0); the MS365 sidebar lists ONLY `surface=ms365`. (spec §6)
- Reuse existing service-client methods `listConversations(query?, surface?)`, `getConversation(id)`, `appendConversationMessage(id, role, text, attachments?, skills?)` — do NOT add new ones. Do NOT touch backend/router/session-scope/connector/Cowork sidebar.
- Renderer never touches the DB or secret bytes; persisted messages are user-visible text only (no token/secret). ESM `.js` imports; `node:test` + `node:assert/strict`.
- Commands: `npm run typecheck` (tsc -b), `npm test`, `scripts\verify-fast.bat`. NOTE: `npm test` has ~16-17 files of KNOWN pre-existing failures + `Merge/` glob noise — confirm NO NEW failures, not a green suite.

export function renderMsAssistant(
  container: HTMLElement,
  view: MicrosoftIntegrationView,
  handlers: MsAssistantHandlers,
  conversations: readonly MsAssistantConversationItem[],
): HTMLElement {
  container.replaceChildren();
  const layout = el("div", "ms-assistant-layout");
  const column = el("div", "ms-assistant");
  const transcript = el("div", "ms-assistant__transcript");
  if (view.connectionState !== "connected") {
    const card = el("section", "ms-card ms-assistant__empty");
    const logo = el("div", "ms-assistant__logo");
    logo.append(createMicrosoftLogo(30));
    const cta = el("button", "ms-assistant__connect-cta", "Mở trang kết nối") as HTMLButtonElement;
    cta.type = "button";
    cta.addEventListener("click", handlers.onOpenConnect);
    card.append(
      logo,
      el("h2", "ms-card__title", "Chưa kết nối Microsoft 365"),
      el("p", "ms-card__copy", "Kết nối tài khoản để trợ lý thao tác trên Outlook, Teams, SharePoint và Planner thay bạn."),
      cta,
    );
    transcript.append(card);
    column.append(transcript, renderComposer(false, handlers.onSend));
    layout.append(column); // no sidebar when disconnected
    container.append(layout);
    return transcript;
  }
  const sidebar = renderMs365Sidebar(conversations, handlers);
  column.append(transcript, renderComposer(true, handlers.onSend));
  layout.append(sidebar, column);
  container.append(layout);
  return transcript;
}

function renderMs365Sidebar(
  conversations: readonly MsAssistantConversationItem[],
  handlers: MsAssistantHandlers,
): HTMLElement {
  const sidebar = el("aside", "ms-history");
  const newBtn = el("button", "ms-history__new", "Cuộc trò chuyện mới") as HTMLButtonElement;
  newBtn.type = "button";
  newBtn.addEventListener("click", () => handlers.onNewConversation());
  const list = el("ul", "ms-history__list");
  for (const conv of conversations) {
    const item = el("li", "ms-history__item");
    const btn = el("button", "ms-history__item-btn", conv.title || "Cuộc trò chuyện") as HTMLButtonElement;
    btn.type = "button";
    btn.addEventListener("click", () => handlers.onSelectConversation(conv.id));
    item.append(btn);
    list.append(item);
  }
  sidebar.append(newBtn, list);
  return sidebar;
}
