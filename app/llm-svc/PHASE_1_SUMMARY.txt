IMPLEMENTATION DETAILS
───────────────────────────────────────────────────────────────────────────────

SERVICE LAYER (T159)
  - Main entry point: src/main.rs
    - Reads LLMSVC_ADDR (default 0.0.0.0:9090)
    - Reads NLP_MODE (default 2)
    - Initializes tracing
    - Creates LlmSvcImpl and starts tonic server

  - RPC Implementation: src/service.rs
    - 8 RPC methods: Embed, Rerank, ExtractEntities, Compress, DetectIntent, Generate, Health, ListModels
    - All methods use Router::route() to decide local vs cloud
    - Cloud proxy integration: CloudProxyClient
    - Error handling: Status::unimplemented (phase 2), Status::unavailable, Status::failed_precondition

ROUTING LOGIC (T164)
  - NlpMode enum: 1 (CloudOnly), 2 (CloudWithLocalPreprocess), 3 (LocalOnly)
  - Router::route(task_name) returns RouteDecision (Cloud | Local | Error)
  - Mode 1: all tasks → Cloud
  - Mode 2: preprocess (detect_intent, query_ner, compress) → Local; other → Cloud (with fallback)
  - Mode 3: all tasks → Local (no fallback)
  - Exponential backoff: max_attempts=3, base_delay=1000ms, max_delay=32000ms, jitter=true

CLOUD PROXY (T163)
  - CloudProxyClient: reqwest-based HTTP client
  - API endpoint: LLM_API_BASE_URL (default https://mkp-api.fptcloud.com/v1)
  - API key: LLM_API_KEY
  - Model: LLM_MODEL (default gpt-4o-mini)
  - Methods:
    - generate(prompt, max_tokens, temperature) → String
    - extract(text) → (Vec<String>, Vec<(String, String)>)
    - detect_intent(query) → String
  - Request format: OpenAI-compatible /chat/completions endpoint
  - Response parsing: extracts choices[0].message.content

CONFIGURATION (T166)
  - Config struct: holds all runtime settings
  - Environment variables:
    - LLMSVC_ADDR (default 0.0.0.0:9090)
    - NLP_MODE (default 2)
    - LLM_API_BASE_URL, LLM_API_KEY, LLM_MODEL
    - BRAIN_LOCAL_PROVIDER, BRAIN_FALLBACK_TO_CLOUD
    - MODELS_YAML_PATH (optional)
  - models.yaml: defines embedding/reranker/generative models
  - ModelRegistry: thread-safe model storage with add/get/list/replace methods

MODELS & INFERENCE STUBS (T160–T162)
  - ModelFormat enum: Onnx, Gguf, Safetensors, Cloud
  - ModelKind enum: Embedding, Reranker, Generative, Other
